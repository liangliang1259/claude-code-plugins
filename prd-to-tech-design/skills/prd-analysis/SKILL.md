# PRD分析技能

此技能提供可复用的PRD分析能力，可从多个上下文中调用。

## 何时使用此技能

当你需要以下操作时使用此技能：
- 分析PRD文档
- 从产品规格中提取需求
- 为技术设计准备结构化输入
- 验证PRD完整性

## 技能能力

### 1. 需求提取
- 识别功能需求
- 识别非功能需求
- 提取用户故事和验收标准
- 将需求映射到功能

### 2. 约束分析
- 技术约束
- 业务约束
- 法规约束
- 预算和时间约束

### 3. 差距分析
- 识别缺失信息
- 标记模糊需求
- 建议所需澄清
- 评估完整性

### 4. 优先级排序
- 按优先级分类（必须有、应该有、最好有）
- 识别依赖关系
- 按逻辑顺序排列需求

## 输入格式

将PRD内容提供为：
- markdown文件的文件路径
- 直接的markdown内容
- 飞书文档ID（如已配置）

## 输出格式

返回JSON格式的结构化分析，包含：
- 需求清单
- 约束总结
- 风险标记
- 建议

## 使用示例

```
/skill use prd-analysis
输入: [PRD内容或文件路径]
重点: [requirements|constraints|risks|all]
```

## 最佳实践

- 保持分析一致性
- 使用标准需求分类
- 提供可追溯性
- 记录假设
- 标记需要澄清的领域
